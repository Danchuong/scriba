Step 1 / 18

Disjoint-Set Union (DSU) on 7 elements. Each node is its own parent initially: parent[x]=x for all x. No parent edges yet. We track parent pointers as a directed edge child→parent in the graph, and as a value in the parent[] array. A find walks child-edges to the root; path compression rewrites the edge so the child points directly at the root. Scriba v0.6 Graph mutation via add_edge/remove_edge lets us show this truthfully.

Step 2 / 18

union(A,B): roots are A and B. Link B under A by adding the directed edge B→A. parent[B]=A. Component: {A,B} with root A. Graph edge count: 0→1.

Step 3 / 18

union(C,D): link D under C: add directed edge D→C. parent[D]=C. Components now: {A,B},{C,D},{E},{F},{G}. Graph edge count: 1→2.

Step 4 / 18

union(E,F): link F under E. Edge F→E, parent[F]=E. Components: {A,B},{C,D},{E,F},{G}. Graph edge count: 2→3.

Step 5 / 18

union(A,C): we need the roots first. find(A)=A (root, no outgoing parent edge). find(C)=C (root). Distinct roots, so we link one tree under the other.

Step 6 / 18

Link root C under root A: add edge C→A, set parent[C]=A. Tree is now A←B, A←C←D. Finding the root of D now requires walking D→C→A: two hops. This is the chain that path compression will collapse next. Graph edge count: 3→4.

Step 7 / 18

find(D) with path compression, step 1: start at D. parent[D]=C, so walk edge D→C. Remember D as a node that will be reparented after we reach the root.

Step 8 / 18

From C: parent[C]=A, so walk edge C→A. C's parent is already the root, so C itself does not need compression.

Step 9 / 18

Reached A: parent[A]=A, this is the root. find(D) returns A. Now compress the path: rewrite D's parent edge so it points directly at A instead of going through C.

Step 10 / 18

Path compression: remove_edge(D→C) then add_edge(D→A). parent[D]=A. Subsequent find(D) calls are now O(1). Graph edge count: 4→3→4 (the remove/add pair). Note: C still points to A. Truthful DSU requires this edge rewrite -- Scriba v0.6 Graph mutation makes it possible.

Step 11 / 18

union(E,G): find(E)=E, find(G)=G. Link G under E: add G→E, parent[G]=E. Component: {E,F,G} rooted at E. Graph edge count: 4→5.

Step 12 / 18

union(A,E): find roots. find(A)=A. find(E)=E. Distinct. Link the smaller tree (rooted at E, size 3) under the larger (rooted at A, size 4).

Step 13 / 18

Link: add edge E→A, parent[E]=A. Now A is the root of everything except G (whose root path is G→E→A) and F (whose path is F→E→A). Both still need compression. Graph edge count: 5→6.

Step 14 / 18

find(F) with path compression, step 1: start at F. parent[F]=E, walk edge F→E.

Step 15 / 18

From E: parent[E]=A, walk edge E→A. E's parent is already the root, no compression needed for E itself.

Step 16 / 18

Reached root A. find(F) returns A. Compress: rewrite F's parent edge to point straight at A.

Step 17 / 18

remove_edge(F→E) then add_edge(F→A). parent[F]=A. Scriba's Graph warm-start layout keeps F in roughly the same visual location -- only the arrow endpoint moves. This is the truthful reparent: the old edge is gone, the new one is in its place. Graph edge count: 6→5→6.

Step 18 / 18

Final state: all seven elements are in one component with root A. parent[]=[A,A,A,A,A,A,E]. Note: G still points to E, not A, because find(G) was never called -- DSU only compresses paths touched by find, which is exactly why the amortized cost is O(α(n)) per operation rather than O(1). A future find(G) would rewrite G→A on the spot. Edge count changed 0→1→2→3→4→3→4→5→6→5→6 via 2 remove/add rewrites -- truthful, not cosmetic.
