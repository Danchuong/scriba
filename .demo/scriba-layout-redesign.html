The widget is too tall because the layout only stacks vertically

An investigation into why embedded Scriba scenes read as a cramped portrait strip — and four ground-up layout directions that trade height for width, shown at honest scale inside a 700 px document column.

Current viewBox

240 × 452 px

Aspect (W : H)

1 : 1.88 portrait

Layout modes that change geometry

1 vertical-stack only

The root cause is structural, not cosmetic. compute_viewbox() in _frame_renderer.py always stacks primitives top-to-bottom: total_height += h + _PRIMITIVE_GAP while width = max(primitive widths). So width is pinned to the single widest primitive (~208 px) while height is the sum of all of them plus two 50 px gaps. Three primitives already give a 1.88 portrait; every primitive you add makes it taller and narrower. The layout: filmstrip | stack option only writes a data-layout attribute — it never touches the geometry. The four directions below all reclaim the empty horizontal space a document column gives you for free.

Recommendation & engineering effort

Ranked by value-for-effort against the Scriba codebase. All four need a horizontal-capable layout pass in compute_viewbox() except D4, which only retunes constants.

DirectionWhat changes in ScribaEng. effortRiskBest when
D4 Refined Tight Stack Shrink _PRIMITIVE_GAP 50→16, inline array captions, render VariableWatch as a horizontal chip row. No new layout engine. Low Low Ship now. Pairs with the upscale fix already applied.
D1 Landscape Filmstrip Add a horizontal pack mode to compute_viewbox (sum widths, max height). Honour the existing layout option. Medium Low ≤3 primitives. Shortest height; best default for embeds.
D3 Linked Pair + Rail Horizontal pack + a "linked pair" relationship between two same-shape arrays + a side rail slot for watches. Medium Medium Comparison algorithms (two-pointer, merge) where pairing is the story.
D2 Bento Grid Full grid layout engine: assign primitives to cells, auto-flow by count/aspect. Largest change. High Medium 4+ primitives / dashboards. Scales best long-term.

Suggested path: ship D4 immediately (constants only, stacks with the max-width upscale fix already in _frame_renderer.py), then build the horizontal pack mode that unlocks D1 as the embed default and D3 for comparison scenes. Keep D2 on the roadmap for when scenes routinely exceed three primitives.
