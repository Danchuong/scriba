## 1. Executive summary

**Severity counts**

| CRITICAL | HIGH | MEDIUM | LOW | NOTE |
|---------:|-----:|-------:|----:|-----:|
| **11** | **18** | **12** | **9** | **4** |

**Top 5 most critical findings**

1. **Stack relies on `slide-in-vertical` / `slide-in-horizontal` keyframes that
   keyframe-animation.md explicitly forbids.** `extensions/keyframe-animation.md`
   §2 declares the preset vocabulary closed to exactly five names (`rotate`,
   `orbit`, `pulse`, `trail`, `fade-loop`) with E1380 for anything else; but
   `primitives/stack.md` §4.1 and §10 routes push animation through the
   `@keyframes` slot using `slide-in-vertical` / `slide-in-horizontal`.
   Implementation will be blocked on day one. Either Stack must avoid the
   keyframe slot, or keyframe-animation must open the vocabulary or add
   slide-in presets.
2. **`\fastforward` + `MetricPlot` wire protocol contradicts itself.**
   `extensions/fastforward.md` §11 declares
   `series=[{name="cost", color=current}, {name="temp", color=done, axis=right}]`
   as list-of-dicts with `axis=right` (a two-axis-chart feature);
   `primitives/metricplot.md` §2.1 declares `series` as `list of strings`, with
   no two-axis support anywhere in the spec. The canonical Problem #9 example
   that supposedly closes the coverage gap cannot even parse against
   `MetricPlot`'s own grammar.
3. **`Matrix` invents a tuple-in-index selector syntax not in the base BNF.**
   `primitives/matrix.md` §4 uses `m.range[(i1,j1):(i2,j2)]`. Base spec §4.1's
   accessor grammar is `"range" "[" index ":" index "]"` where `index ::= NUMBER
   | INTERP`. Parenthesized tuples are not a base-spec index. Matrix does not
   flag this as a base-spec delta in §14. The selector parser as spec'd in Phase
   A1 cannot accept this input.
4. **`Graph layout=stable` cache key is wrong.** `graph-stable-layout.md` §5.2
   hashes `sorted_union_edges`, but the SA objective in §4.3 is `sum_t
   edge_crossings(E_t, ...)` — depends on the **per-frame edge sets**, not just
   the union. Two animations with identical union but different frame orderings
   will cache-collide and render with wrong positions.
5. **Renderer `priority` value contradicted across three documents.** Base spec
   §10.2 locks `AnimationRenderer.priority = 10` (and earlier = higher priority,
   first-wins). Both `04-roadmap.md` Phase A deliverables and
   `05-implementation-phases.md` Week A1 say `priority = 50 (lower than
   TexRenderer default 100)`. Implementers will have to guess which is
   canonical. figure-embed.md §11 says 10 again, which contradicts the roadmap
   it was supposed to merge into.

**Go / no-go recommendation: NO-GO for Wave 0 code agents as of now.** The spec
set is roughly 80% implementation-ready but contains six contradictions that
will cause two code agents to produce incompatible outputs. Fix the 11 CRITICAL
and the top-tier HIGH findings first (roughly 4–6 hours of focused edit work)
and then start coding. The effort is small; leaving them will cost 1–2 weeks of
rework during Phase A and Phase C when the contradictions surface in CI.
