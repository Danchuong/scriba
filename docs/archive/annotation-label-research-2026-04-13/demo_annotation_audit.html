Key finding: 11/14 primitives support arrow annotations, ALL go through shared emit_arrow_svg() in base.py. Non-arrow annotations (position=above/below) are parsed and stored but never rendered to SVG.

1. Array: 3 arrows to same cell

dp[3] ← dp[0] "+3", dp[1] "+4", dp[2] "+1". Stagger = vertical only.

Current labels overlap, no background

Proposed pills + staggered

2. Array: long label overflow

Label: "min(dp[i-1]+cost(i-1,i), dp[i-2]+cost(i-2,i))" = 46 chars

Current overflows SVG width

Proposed truncated 20 chars + pill

3. DPTable 2D: same-column + same-row arrows

dp[0][1] ← dp[0][0] (same row), dp[1][1] (same col). Labels collide.

Current vertical degenerate + label collision

Proposed arc + pills + nudge
