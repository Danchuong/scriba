Annotation Stress Test: Complex Real-World Cases

Worst-case scenarios from actual CP editorials. Each shows Current (problems) vs Proposed (fix).

1. Knapsack DP: 5 arrows into dp[4], varying span lengths

dp[4] = min(dp[0]+8, dp[1]+3, dp[2]+6, dp[3]+1, dp[3]+2). Items from multiple predecessors with cost labels. The most common DP editorial pattern.

Current 5 labels pile up in same Y band, unreadable

Proposed pills + collision nudge + stagger

2. Interval DP (2D): 6 arrows crossing in dp[0][4]

dp[0][4] depends on dp[0][k] + dp[k+1][4] for k=0..3. 4 pairs of arrows (8 total) converge from different rows and columns. The densest 2D annotation scenario.

Current 8 arrows, labels form a wall of unreadable text

Proposed show only best k, dim others, pills

3. Dijkstra: 4 edges into node, with distance labels

Node E reachable from A(d=10), B(d=7), C(d=12), D(d=5). Each edge has "d=X via Y" label. Graph layout with 2D perpendicular arrows.

Current 4 labels collide around node E

Proposed mid-path pills, only best labeled, dim others

4. BFS Tree: hidden-state reveal with annotations

Tree grows as BFS discovers nodes. Annotations "d+1" on each newly revealed edge. 5 annotations visible simultaneously.

Current labels overlap tree edges, no background

Proposed pills + offset from edge midpoint

5. LIS traceback: reannotated path with long labels

8-cell array, optimal path highlighted. Each path arrow gets "dp[j]+1=dp[i]" label. 4 path arrows + 6 dimmed arrows all visible at once.

Current 10 labels, path labels overlap dimmed labels

Proposed path pills prominent, dimmed labels hidden
