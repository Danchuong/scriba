Annotation Label Placement Strategies

Comparing approaches for labeling DP transition arrows on array visualizations. Data: array [2, 9, 4, 5, 1, 6] with three arrows pointing to cell[3] from cells [0], [1], [2].

1. Current Approach — Centered Above Bezier Peak

Problem

Labels are placed at the horizontal midpoint of the arrow, vertically centered above the Bezier peak. When multiple arrows converge on the same target, labels stack on top of each other at similar heights, creating unreadable overlap.

Problems

  • Labels collide when arrows converge on one cell
  • Hard to tell which label belongs to which arrow
  • Gets worse as more arrows are added

2. Strategy: Mid-Path Labels

Strategy

Place each label at the geometric midpoint of its own Bezier curve, rotated to follow the tangent angle. Each label sits on its own curve, naturally spreading them apart.

Pros

  • Labels are naturally separated by curve geometry
  • Clear which label belongs to which arrow
  • Rotation adds visual association with the path

Cons

  • Rotated text can be harder to read
  • Labels on short/flat arcs may still overlap
  • Implementation requires computing Bezier midpoint + tangent

3. Strategy: Color-Only (No Text Labels)

Strategy

Remove text labels entirely. Distinguish arrows by color and rely on a separate legend below the diagram. The visual is much cleaner, especially with many arrows.

Pros

  • Cleanest visual; zero text clutter above the array
  • Scales well to many arrows
  • Easy to implement

Cons

  • Loses per-arrow detail (exact values only in legend)
  • Color alone is not accessible (color-blind users)
  • Requires scanning back and forth between diagram and legend

4. Strategy: Numbered Badges

Strategy

Replace verbose text with small numbered circles on each arrow. A legend below maps each number to its full description. Compact yet informative.

Pros

  • Compact badges never overlap (small footprint)
  • Full descriptions available in legend
  • Works well even with many arrows

Cons

  • Requires scanning between badge and legend
  • Numbers alone carry no semantic meaning in-context
  • Legend can grow long with many arrows

5. Strategy: Background Pills

Strategy

Add a rounded rectangle (pill) behind each label, similar to GitHub PR labels or badges. The opaque background prevents text from blending into crossing arrows. Labels are offset vertically to avoid stacking.

Pros

  • Opaque background ensures readability over crossing lines
  • Labels stay inline on their arrows
  • Familiar visual metaphor (badge / tag)

Cons

  • Pills can obscure the arrow path beneath them
  • Still need vertical staggering to prevent overlap
  • Adds visual weight; busy with many arrows

6. Strategy: Stacked Label Column with Leader Lines

Strategy

All labels are collected into a vertical column to the right of the array. Thin leader lines connect each label to the midpoint of its corresponding arrow. The arrows themselves remain clean and unlabeled.

Pros

  • Zero overlap; labels are always vertically stacked with uniform spacing
  • Arrows stay clean and uncluttered
  • Supports long label text without crowding the diagram
  • Leader lines maintain arrow-label association

Cons

  • Requires extra horizontal space for the column
  • Leader lines can cross each other with many arrows
  • Visual indirection: eye must follow leader line to connect label to arrow
