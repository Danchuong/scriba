## 5. New Contradictions Introduced

No HIGH or CRITICAL contradictions were introduced by the fix pass. The following
LOW issues were created or remain:

**L5.1 — `primitives/SANITIZER-WHITELIST-DELTA.md` not cited in Week C5 task bullet.**
`05-implementation-phases.md` Week C5 explicitly names only the extensions whitelist
file. An implementer reading Week C5 might skip the primitives delta. Severity: LOW.
Both files exist and are internally consistent; this is a task-description gap.

**L5.2 — `data-layout` attribute in `graph-stable-layout.md` uses bare `data-layout` (not `data-scriba-layout`).**
The primitives SANITIZER-WHITELIST-DELTA.md acknowledges this at the bottom of the
data-attributes section with a forward-looking note about future consolidation. Not a
new contradiction — acknowledged and deferred.

**L5.3 — `stack.md` §10 says animations ship via `required_css` in `scriba-stack.css` but `extensions/keyframe-animation.md` §4.2 says they ship via `required_css["keyframes/scriba-keyframes.css"]`.**
Both files say the CSS ships via `required_css` — the filenames differ (stack implies
`scriba-stack.css` owns the slide-in rules, keyframe-animation says `scriba-keyframes.css`
owns all 7 presets). This is a LOW ambiguity about which static file ships the
slide-in rules. The keyframe-animation spec (§5) is the canonical source and explicitly
includes slide-in-vertical and slide-in-horizontal CSS blocks. Stack §10 should say
"animations defined in `scriba/animation/static/scriba-keyframes.css`" rather than
"live in `scriba-stack.css`". Severity: LOW (implementer will find the canonical CSS
in keyframe-animation.md; the test will make the right file obvious).

**No new CRITICAL or HIGH contradictions.**
